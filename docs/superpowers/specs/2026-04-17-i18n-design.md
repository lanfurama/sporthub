# i18n Multi-language Support вҖ” Design Spec

## Overview
Add 5-language support (English, Korean, Japanese, Vietnamese, Russian) to the customer-facing interface of SportHub.

## Scope
**In scope (customer-facing only):**
- LoginPage, RegisterPage, HomePage, BookingFlow, BookingSuccessPage
- Navbar, PriceBreakdown, MemberSearch, TimeSlotGrid, Badge, Spinner, Modal
- LanguageSwitcher component (new)

**Out of scope:**
- Admin pages (DashboardPage, BookingsPage, AdminBookPage, MembersPage, ProductsPage, OrdersPage)
- Admin components (AdminSidebar, AdminLayout)
- API error messages from server

Default: English (`en`)

## Technology
- `react-i18next` + `i18next` for translation management
- Language stored in URL path prefix (`/:lang/`)

## Routing Strategy
- Customer routes: `/:lang/`, `/:lang/booking`, `/:lang/login`, `/:lang/register`, `/:lang/booking-success`
- Admin routes: unchanged (`/admin/...`)
- `/` redirects to `/en/`
- Invalid lang codes redirect to `/en/` equivalent

## LanguageSwitcher Component
- Location: Navbar, right side, before login/avatar button
- Appearance: Dropdown with flag emoji + native language name
- Behavior: On select, navigate to same path with new lang prefix
- Flags: рҹҮәрҹҮё рҹҮ°рҹҮ· рҹҮҜрҹҮө рҹҮ»рҹҮі рҹҮ·рҹҮә

## Implementation Notes
- Extract all hardcoded strings from customer pages/components into translation keys
- Use `useTranslation()` hook in each component
- Use `useParams()` to read current lang from URL
- Sync i18next language with URL param via a wrapper/layout component
- Date formatting: use `date-fns` locale matching current language
- Number/currency formatting: use `Intl.NumberFormat` with appropriate locale
- Keep brand name "SPORTHUB" untranslated across all languages
